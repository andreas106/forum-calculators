STANDING RULES (saved as memory feedback files)
-----------------------------------------------
1. Every calc MUST include an SEO explainer block inside BEGIN/END markers:
   intro / worked examples / key concept callout / format reference.
   See `feedback_calc_explainer_section.md`.

2. NO currency picker. Display bare unit-less monetary values (`250.00`).
   Label-only currency picker was tried 2026-05-20 and removed because it
   was misleading without actual FX conversion. See `feedback_calc_no_currency.md`.

3. Worked-examples tables MUST be computed via the calc's actual code
   (node -e or live UI), not hand math with rounded intermediates. Caught
   a 2× error in Parlay 10-leg + 4/6 wrong rows in Streak before shipping.
   See `feedback_calc_table_values.md`.

4. When drafting handoff messages to Anna — frame the explainer as
   "turns the calc from a tool into an SEO asset" (verbatim phrase).
   Do NOT mention currency picker (was removed).

Each folder contains `<name>.html` + `<name>.js`. JS imports `../odds-math.js`
when needed (most do; french-roulette, baccarat, and casino-bonus-playthrough
are self-contained).

LOCAL DEV
---------
Server is started via `python3 -m http.server 8765 --directory ./calculators`.
URLs:
  http://localhost:8765/odds-converter/odds-converter.html
  http://localhost:8765/bet/bet.html
  http://localhost:8765/parlay/parlay.html
  http://localhost:8765/sports-futures/sports-futures.html
  http://localhost:8765/streak/streak.html
  http://localhost:8765/french-roulette/french-roulette.html
  http://localhost:8765/baccarat/baccarat.html
  http://localhost:8765/casino-bonus-playthrough/casino-bonus-playthrough.html

ES module imports require HTTP (not file://) — browsers block `import` from
file URLs. Always test via the localhost server.
